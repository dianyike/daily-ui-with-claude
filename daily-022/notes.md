### Round 2 — token drift bulk fix
audit.js 初次跑出 149 P1 unbound-fill drift。三波修：
1. **clearedDefaultWhite: 50** — `figma.createAutoLayout()` 預設給每個 frame 一個 white solid fill，inner layout containers (HeaderRow / *Cluster / *Block / *Col / *Row) 應該透明繼承 section BG。Bulk clear: any frame with single unbound `{type: SOLID, color: {r:1,g:1,b:1}, opacity: 1}` → fills = []
2. **rebindInkOpacity: 70** — text nodes 用 literal `{r: 0.102, g: 0.102, b: 0.102, opacity: 0.X}` 做 typographic hierarchy tier (0.6 / 0.45 / 0.35 / 0.55 etc.) → 改為 bound to `scene/ink-stamp` + node-level opacity（per F11 trap: paint-level opacity resets to 1 on bound color）
3. **silRebind: 5 + gridRebind: 12 + trackRebind: 1** — 加 `scene/silhouette-graphite` token、bind 5 victim Silhouettes / 12 grid lines (node opacity) / 1 confidence bar track (node opacity)

最終剩 11 P1 drift = 11 個 S4 zebra-stripe row 背景 paint.opacity 0.025。Row 是 auto-layout 父含 text 子，**node.opacity 會傳到 children** 把 text 也變 2.5% 不可見。`bound color + paint.opacity` 受 F11 trap 阻擋。唯一無代價解法是改 row 結構為「row=透明 wrapper + 內部 bg-rect 0.025」、但這個結構為 cosmetic gain 而做不值得。**Documented intentional drift**（zebra stripe 是 tabular hierarchy 必要視覺輔助 + 不影響 codegen 因為實際 CSS 用 `background: rgba(26,26,26,0.025)` 直接落地）。
